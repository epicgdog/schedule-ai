# Product Guidelines: Schedule AI

## Branding & Visual Identity
- **Color Palette:** Modern High-Contrast. Use a clean, professional palette with high-contrast colors (e.g., Deep Navy, Slate Gray, Vibrant Gold) to ensure readability and accessibility across all devices.
- **Glassmorphism Aesthetic:** Utilize semi-transparent backgrounds with `backdrop-blur` for panels and interactive elements to create depth and focus.
- **Typography:** Sans-serif fonts (e.g., Inter, Roboto) for a modern, clean look that prioritizes legibility in complex schedules.

## Voice & Tone
- **Style:** Academic and Direct. Use clear, concise language that helps students make informed decisions about their academic path without unnecessary fluff.
- **Tone:** Professional, objective, and supportive. The application should act as a reliable advisor, providing accurate information and clear guidance.

## UX Principles
- **Visual-First Navigation:** Prioritize visual representations of data (e.g., prerequisite trees, schedule grids) over text-heavy catalogs to reduce cognitive load and improve understanding.
- **Step-by-Step Guidance:** Break down complex tasks (like building a full semester schedule) into manageable steps, guiding the student through each phase of the process.

## Interaction Design
- **Real-Time Feedback:** Provide immediate visual feedback when a course is added or a constraint (like a time conflict or prerequisite violation) is encountered.
- **Micro-Interactions:** Use subtle animations and transitions to indicate state changes (e.g., loading data, expanding a course detail) to create a more engaging and responsive experience.
- **Accessibility:** Adhere to WCAG 2.1 AA standards to ensure that Schedule AI is usable by all SJSU students, including those with disabilities.
